Sticky Navigation
dengan Scroll Behavior

Pola navigasi yang tetap terlihat saat pengguna menggulir halaman. Scroll ke bawah untuk melihat demo langsung dari berbagai varian!

Scroll ke bawah

01 — Pengenalan

Apa itu Sticky Navigation?

Sticky Navigation adalah pola UI di mana header atau navbar tetap terlihat di bagian atas viewport saat pengguna menggulir konten ke bawah. Pattern ini menyelesaikan masalah fundamental: pengguna yang sudah scroll jauh ke bawah masih bisa mengakses navigasi tanpa harus kembali ke atas.

Penelitian dari Nielsen Norman Group menemukan bahwa sticky headers meningkatkan kecepatan navigasi rata-rata 22% — pengguna bisa berpindah halaman lebih cepat karena navigasi selalu dalam jangkauan.

💡

Tips: Scroll Sekarang!

Coba scroll ke bawah dan perhatikan bagaimana navbar di atas berubah. Gunakan panel kontrol di pojok kanan bawah untuk mencoba berbagai varian behavior.

02 — Varian

4 Varian Utama

Klik kartu untuk mengaktifkan behavior tersebut secara langsung pada demo ini:

🎭

Hide / Show

Sembunyi saat scroll bawah, muncul saat scroll atas. Digunakan oleh Medium & Airbnb.

📌

Always Visible

Selalu terlihat di atas. Cocok untuk web app dengan banyak navigasi. Digunakan GitHub.

🔎

Shrink on Scroll

Navbar mengecil setelah melewati threshold. Digunakan Apple & Stripe.

🌅

Progressive Sticky

Muncul setelah melewati hero section. Digunakan Notion & Figma.

03 — Implementasi

Kode di Balik Magic-nya

Implementasi terbaik menggunakan kombinasi CSS untuk animasi (GPU-accelerated dengan transform) dan JavaScript untuk logika scroll. Selalu gunakan requestAnimationFrame untuk menghindari jank.

// ✅ Implementasi yang benar — performan & accessible class StickyNav { constructor(navElement) { this.nav = navElement; this.lastScrollY = 0; this.threshold = 80; this.ticking = false; this.init(); } init() { // passive:true = tidak memblok rendering window.addEventListener('scroll', () => this.onScroll(), { passive: true }); } onScroll() { // rAF = batching update di frame rate optimal if (!this.ticking) { requestAnimationFrame(() => { this.update(); this.ticking = false; }); this.ticking = true; } } update() { const y = window.scrollY; // Tambah shadow setelah scroll melewati threshold this.nav.classList.toggle('scrolled', y > this.threshold); // Hide saat scroll bawah, show saat scroll atas const goingDown = y > this.lastScrollY && y > this.threshold; this.nav.classList.toggle('hidden', goingDown); this.nav.setAttribute('aria-hidden', goingDown); this.lastScrollY = y; } }
/* Wajib: agar anchor link tidak tertutup navbar */ html { scroll-padding-top: 70px; } /* Gunakan transform, BUKAN top/height — lebih performan */ .navbar.hidden { transform: translateY(-100%); visibility: hidden; /* blok tab focus */ } /* Wajib untuk inklusivitas */ @media (prefers-reduced-motion: reduce) { .navbar { transition: none !important; } }
⚠️

Jangan Animasikan `top` atau `height`

Property tersebut menyebabkan layout reflow yang mahal. Selalu gunakan transform: translateY() yang berjalan di GPU compositor thread.

04 — Aksesibilitas

Inklusif untuk Semua Pengguna

Sticky navigation yang tidak diimplementasikan dengan benar bisa menjadi hambatan bagi pengguna disabilitas. Berikut hal-hal kritis yang wajib diperhatikan:

♿

WCAG 2.4.1 — Skip Navigation

Sediakan "Lewati ke konten utama" yang muncul saat difokus keyboard. Di demo ini, tekan Tab pertama kali untuk melihatnya muncul di pojok kiri atas!

Checklist Aksesibilitas

✅
Skip Navigation Link

Tersedia di demo ini. Tekan Tab untuk melihatnya.

✅
visibility: hidden saat navbar tersembunyi

Mencegah link tersembunyi tetap bisa difokus keyboard.

✅
aria-hidden otomatis saat navbar tersembunyi

Screen reader tidak akan mengumumkan navigasi tersembunyi.

✅
prefers-reduced-motion

Semua animasi dinonaktifkan untuk pengguna sensitif gerak.

✅
scroll-padding-top

Anchor link tidak tertutup oleh navbar yang posisinya fixed.

✅
Kontras warna minimum 4.5:1

Semua teks navigasi memenuhi WCAG AA standard.

05 — Contoh Nyata

Siapa yang Menggunakannya?

Hampir semua produk web terkemuka menggunakan sticky navigation — tapi masing-masing dengan pendekatan berbeda:

✈

Airbnb

Hide/Show

Search bar di navbar bertransformasi saat scroll — mengecil dari form besar menjadi pill kecil. Masterpiece scroll behavior!

M

Medium

Progressive

Transparan di hero, solid setelah melewatinya. Menambahkan reading progress bar terintegrasi di dalam navbar.

🐙

GitHub

Always Visible

Navbar utama selalu ada, ditambah sticky sub-nav per halaman repo. Pilihan aman untuk web app kompleks.

▼

Stripe

Shrink+Glass

Navbar mengecil saat scroll dengan efek frosted glass. Background berubah dari transparan menjadi blur.

N

Notion

Progressive

Navbar tersembunyi di halaman marketing, muncul hanya setelah pengguna melewati hero section pertama.

🎨

Figma

Progressive

Sticky nav yang muncul saat scroll dengan CTA "Get started free" yang selalu terlihat untuk mendorong konversi.
